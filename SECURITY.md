# Security Policy

## Reporting A Vulnerability

Please do not open a public issue for security-sensitive problems.

Instead, contact the maintainer privately and include:

- A description of the issue
- Reproduction steps
- Impact
- Suggested fix, if you have one

## Scope

This project combines local file processing with a remote Gemini API call. Security reports are especially useful for:

- Unsafe file handling
- Unexpected file deletion behavior
- Path handling issues
- Secret loading and secret leakage risks
- Dependency-related risks

## Response

Security reports will be reviewed and addressed as quickly as possible.
